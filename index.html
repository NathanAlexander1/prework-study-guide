Git

  • Git is a version control system that lets you manage and keep track of each version of your project.
  • git status: checks what branch we are currently on
  • git checkout branch-name: swtiches branches
  • git checkout -b branch-name: creates a new branch and switches to it
  • To merge a working branch to a main branch:
    1. Once I have made changes to new branch, make sure I am in that branch in my local environment
    2. git add -A: sets up all changes to be committed
    3. git commit -m "message describing the change": this captures a snapshot of the previously staged changes
    4. git pull origin main: verifies that the local branch is in sync with the main branch in GitHub
    5. git push origin name-of-branch-being-merged: Once verified, we can push the changes from the local branch into the remote
    6. Go to remote branch. In the dropdown menu, go to the new branch. Select the "Compare and Pull Request" button. Give the pull request a decriptive title. Designate base branch as "main" and comparison branch as whatever the new branch is titled. Then click "Create Pull Request"
