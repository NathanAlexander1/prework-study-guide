HTML

  • HTML uses the id attribute as a way to identify elements and select them
  • The head element contains information about the webpage (metadata).
  • Metadata is not seen by viewers
  • The body element represents the visible content shown to the user.
  • HTML elements can include text, image, header, list elements and more
  • We using heading elements in descending order to give our webpage structure

CSS

  • * applies CSS styling to ALL elements on a page
  • To link a CSS file to an HTML: < link rel="stylesheet" href="pathway of css file" >
  • A margin indicates how much space we want around the outside of an element (like margins in a word doc, if set to 0, text can run to the very end of the page).
  • A padding indicates how much space we want around the content inside an element (essentially, makes that element bigger).
  • Class attributes - allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector. Then the class attribute can be assigned to the element in the HTML, enabling us to freely choose which elements will be assigned the CSS rule
  • Remember that we can style HTML elements using the elements as selectors or by using the class attribute as a selector.

Git

  • Git is a version control system that lets you manage and keep track of each version of your project.
  • git status: checks what branch we are currently on
  • git checkout branch-name: swtiches branches
  • git checkout -b branch-name: creates a new branch and switches to it
  • To merge a working branch to a main branch:
    1. Once I have made changes to new branch, make sure I am in that branch in my local environment
    2. git add -A: sets up all changes to be committed
    3. git commit -m "message describing the change": this captures a snapshot of the previously staged changes
    4. git pull origin main: verifies that the local branch is in sync with the main branch in GitHub
    5. git push origin name-of-branch-being-merged: Once verified, we can push the changes from the local branch into the remote
    6. Go to remote branch. In the dropdown menu, go to the new branch. Select the "Compare and Pull Request" button. Give the pull request a decriptive title. Designate base branch as "main" and comparison branch as whatever the new branch is titled. Then click "Create Pull Request"

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • A variable can hold several types of data:
    1. String - a sequence of text inside of quotes
    2. Number - this is a number and doesn't need quotes around it
    3. Boolean - this is a true/false value
    4. Array - This is a structure that allows you to store multiple values in a single reference. These values can be different data types. Arrays are enclosed inside square brackets
    5. Objects - This can be anything. Everything in JavaScript is an object and can be stored in a variable.
  • Control flow is the order in which a computer executes code in a script. The computer reads JS from top to bottom. Using conditional statements such as "if" statements allows us to interrupt control flow
  • Conditionals are code structures used to test if an expression returns true or not. These can be "if" statements or "if...else" statements. Conditionals test whether or not the condition provided is truthy. If it is, then the condition is met and an action is executed
  • For loops - use the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over. Developers must use another statement, such as the conditional if statement, to interrupt the control flow by executing a block of code a specific number of times.
  • Functions - a way of packaging functionality that you wish to reuse. It's possible to define a body of code as a function that executes when you call the function name in your code. This is a good alternative to repeatedly writing the same code. Usually look like a variable name followed by parentheses. Functions often take arguments (bits of data they need to do their job. Arguments go inside the parentheses, separated by commas if there is more than one argument).
  • Events - Real interactivity on a website requires event handlers. These are code structures that listen for activity in the browser, and run code in response. For example, a click event
  • ChromeDevTools - allows you to test code more easily by viewing the console. Use "control + shift + i" to access
